**功能分支**：`001-ai-resource-platform`  
**创建日期**：2025-02-13  
**状态**：草稿  
**输入**：来自 SPEC_KIT_FEED.md（第 54–78 行）的用户描述：公司内部平台，用于分享 Skill/Rule 并学习 AI 概念与资讯。

## 澄清说明

### 2025-02-13 会话

- Q: 管理员能否删除或隐藏 Skill/Rule？ → A: 可以。管理员可以删除或隐藏任意 Skill/Rule（例如违规或过期内容）。Skill 与 Rule 的创建、编辑、删除均在管理后台完成，前台无「上传」「我的上传」。
- Q: 分类与标签是预置还是可由管理员自建？ → A: **分类**由种子数据预置，不在管理后台维护；**标签**由管理员在管理后台维护；创建 Skill/Rule 时从现有分类与标签中选择。
- Q: 用户能否看到「谁」创建的？能否按创建者筛选或搜索？ → A: 仅展示创建者（管理员）。列表/详情显示创建者；不支持按创建者筛选或搜索（筛选/搜索仅关键词、分类、标签）。
- Q: Skill 与 Rule 的区别？ → A: **Rule** 仅支持纯文本（名称、描述、分类、标签、正文），内容存于平台。**内部 Skill** 不存正文与附件，仅在平台登记元数据与 **clone 命令**（内容在外部 GitLab 维护）；**外部 Skill** 仅存元数据与安装命令。
- Q: Skill 分两种（外部 skills.sh 爬取 vs 公司内部创建），应如何设计？ → A: **推荐方式**：**内部 Skill** 使用 **Skill** 表，登记 clone 命令（内容在 GitLab）；不在此上传文件。「**外部 Skill**」单独建表（如 ExternalSkill），仅存**元数据**：名称、描述、安装命令（如 `npx skills add owner/repo`）、来源 URL（如 [skills.sh](https://skills.sh/) 链接）；**不爬取并存储完整技能内容**。录入方式二选一或并存：**爬取**（定时/手动从 skills.sh 抓取列表写入）、**手工添加**（管理员填写安装命令与链接）。列表/首页可合并展示并带「来源：内部/外部」标识；外部 Skill 详情页展示「安装」按钮与安装命令，用户复制命令或跳转来源链接安装。
- Q: 内部 Skill 如何做版本与多文件维护？ → A: **内部 Skill 改为在系统外部用 GitLab 仓库维护**（多文件、版本由 Git 管理）。平台**不提供文件上传**，仅存储并展示每条 Skill 的 **clone 命令**（如 `git clone <GitLab 仓库 URL>`）；用户通过复制 clone 命令在本地获取完整 Skill 内容。内部 Skill 表仅存元数据（名称、描述、分类、标签、**clone_command**、登记人、可见性），不存正文与附件。
- Q: 用户账号创建方式（v1）？ → A: **仅自注册**。提供注册接口，无需审批，注册即可登录；v1 无管理员创建账号流程。
- Q: 搜索关键词最大长度与过长时的处理？ → A: **最大 200 字符**；超长拒绝，返回 400 并提示「关键词过长」。
- Q: 首页推荐位 v1 是否实现？ → A: **v1 不实现**。首页仅展示「最新 N 条」Skill、Rule、文章、资讯，无推荐位与 Recommendation 表；后续版本可增加。
- Q: 内部 Skill 的 clone_command 是否做格式校验？ → A: **不校验格式**。仅校验非空及长度上限（如 2000 字符），超长拒绝并提示。
- Q: 搜索时空关键词的处理？ → A: **空关键词视为未筛选**：忽略 keyword，返回按分页的列表（与不传 keyword 一致）。

## 用户场景与测试 *（必填）*

### 用户故事 1 - 未登录浏览与使用全部内容（优先级：P1）

任何人（包括未认证访客）可打开平台，浏览 Skill 与 Rule 列表，按分类、标签和关键词搜索与筛选，打开任意条目的详情页，并复制或下载内容。也可阅读学习文章与资讯、查看首页。以上操作均无需登录。

**优先级理由**：立即交付价值，并定义大多数用户的默认体验。

**独立测试**：在无痕或已登出会话中打开应用；执行搜索、筛选、打开详情、复制内容，以及打开学习文章与资讯列表/详情。所有操作在未提示登录的情况下完成。

**验收场景**：

1. **假设** 访客未登录，**当** 其打开首页，**则** 看到聚合的最新 Skill、Rule、学习文章与资讯。
2. **假设** 访客，**当** 其打开 Skill 或 Rule 列表，**则** 看到包含名称、描述、分类、创建者、时间的卡片；可按分类/标签筛选、按关键词搜索。
3. **假设** 访客在 Skill 或 Rule 详情页，**当** 使用「复制」或「下载」，**则** Rule 正文被复制或下载；**若为内部 Skill**，**则** 展示 clone 命令供复制（内容在 GitLab，用户自行 clone）；**若为外部 Skill**，**则** 展示安装命令供复制或跳转。
4. **假设** 访客，**当** 打开学习文章或资讯，**则** 无需登录即可看到列表与详情。

---

### 用户故事 2 - 管理员管理 Skill、Rule、标签、文章与资讯（优先级：P2）

Skill 与 Rule 的**创建、编辑、删除**均在管理后台完成。管理员可**登记内部 Skill**（名称、描述、分类、标签、clone 命令）、**创建 Rule**（名称、描述、分类、标签、正文）、管理标签，以及创建与编辑学习文章与资讯。分类由种子数据预置，**不在管理后台维护**。前台**不提供**「上传」或「我的上传」入口；所有用户（含未认证）仅可浏览、搜索、查看详情与复制内容。

**优先级理由**：内容由管理员集中维护，前台纯消费。

**独立测试**：以管理员身份登录管理后台；创建 Skill（填写 clone 命令）与 Rule；确认二者出现在前台公开列表。未认证访客打开前台，确认无「上传」「我的上传」入口；仅可浏览、搜索、查看详情、复制。

**验收场景**：

1. **假设** 管理员，**当** 在管理后台提交新的 Skill（名称、描述、分类、标签、clone 命令）或 Rule（名称、描述、分类、标签、正文），**则** 条目被保存并立即出现在前台公开列表。
2. **假设** 管理员，**当** 在管理后台编辑或删除 Skill/Rule，**则** 变更即时反映在前台列表与详情中。
3. **假设** 管理员，**当** 创建或更新标签，**则** 这些是 Skill/Rule 创建时可选的标签来源；分类由种子数据预置，创建时从现有分类中选择。
4. **假设** 管理员，**当** 发布或编辑学习文章与资讯，**则** 所有访客（无需登录）在对应列表与详情视图中可见该内容。
5. **假设** 已登录的非管理员用户或未认证访客，**当** 打开前台，**则** 不显示「上传」或「我的上传」入口；**假设** 其尝试访问仅限管理员的接口或界面，**则** 系统拒绝访问（如 403 或重定向）。

---

### 用户故事 3 - 管理员隐藏或删除不当内容（优先级：P3）

管理员可删除或隐藏任意 Skill 或 Rule（如不当或过期内容）；被隐藏或删除的条目不再在公开列表中显示。**v1 无首页推荐位**（仅展示最新 N 条）。管理操作需登录且仅限管理员角色。

**独立测试**：以管理员身份登录；隐藏或删除某条 Skill/Rule；确认该条目在前台不可见。确认非管理员无法访问管理后台。

**验收场景**：

1. **假设** 管理员，**当** 隐藏某 Skill/Rule，**则** 该条目不再在前台列表或详情中显示；可由管理员恢复可见性。
2. **假设** 管理员，**当** 删除某 Skill/Rule，**则** 该条目被永久移除，不再在前台展示。

---

### 边界情况

- 空状态：当没有 Skill 或 Rule（或当前搜索/筛选无结果）时，列表显示合适的空状态；不报错。
- 搜索与筛选：列表至少支持关键词及分类/标签筛选（无创建者筛选或创建者搜索）；创建者仅用于展示。**关键词最大 200 字符**，超长则拒绝并返回 400 及提示「关键词过长」。**空关键词视为未筛选**，忽略 keyword 返回分页列表；对特殊字符有定义行为并给出明确反馈。
- 重名或相似名称：多条条目可同名或相似；以稳定 id 区分；列表与详情使用同一 id 保持一致。
- 并发编辑：若管理员在后台编辑某条时另一访客正在查看，下次加载显示更新后内容；v1 不要求实时同步。
- 学习文章与资讯：若无文章或资讯，对应区块或页面显示空或占位状态。

## 需求 *（必填）*

### 功能需求

- **FR-001**：系统**必须**允许未认证用户对全部 Skill 与 Rule 进行列表、搜索、筛选、查看详情及复制或下载，并对学习文章与资讯及首页进行列表与查看。前台**不得**提供「上传」或「我的上传」入口。
- **FR-002**：系统**必须**仅对管理后台要求认证（账号与密码）；前台浏览无需登录。
- **FR-003**：系统**必须**允许管理员在管理后台创建 **Rule**（名称、描述、分类、标签、正文，纯文本）或**登记内部 Skill**（名称、描述、分类、标签、**clone 命令**；Skill 内容与多文件在外部 GitLab 维护，平台不存储正文与附件）。条目**必须**立即出现在前台公开列表（无审批流程）。
- **FR-004**：系统**必须**允许管理员在管理后台编辑与删除 Skill 与 Rule；前台用户**不得**有上传、编辑或删除入口。
- **FR-005**：系统**必须**支持 Skill 与 Rule 的列表与详情，支持按分类与标签筛选、按关键词搜索；结果**必须**分页。列表与详情**必须**展示创建者身份（仅内部）；筛选与搜索**不得**包含创建者（v1 无按创建者筛选或搜索）。**可以**在同一列表或首页中同时展示**内部 Skill/Rule** 与**外部 Skill**（来自 skills.sh 等），并标识来源（内部/外部）。
- **FR-006**：系统**必须**用**两个独立表**分别承载**内部 Skill** 与 **Rule**。**Rule 表**：仅正文（纯文本），无附件，内容存于平台。**Skill 表**：仅存元数据（名称、描述、分类、标签、**clone_command**、创建者、可见性）；**不存正文与附件**，内容与多文件在外部 GitLab 维护，平台仅提供 clone 命令供用户复制。**外部 Skill** 使用独立数据模型，仅存元数据（安装命令、来源 URL 等）。面向用户的界面**可以**通过不同标签页或入口展示。
- **FR-007**：系统**必须**允许管理员在管理后台管理标签；创建与更新 Skill、Rule、学习文章（标题、摘要、富文本正文、分类）与资讯（标题、摘要、来源 URL、日期）。管理员**可以**添加、编辑、删除**外部 Skill**（元数据与安装命令）。**分类由种子数据预置**，不在管理后台提供增删改。
- **FR-007b**：系统**必须**允许管理员删除或隐藏任意 Skill 或 Rule；被隐藏或删除的条目**不得**在公开列表或详情中出现。「隐藏」**可以**实现为软删除或可见性标志；「删除」**可以**为永久移除。
- **FR-008**：系统**必须**以列表与详情形式向所有用户（含未认证）开放学习文章与资讯；资讯**可以**按日期分组。
- **FR-009**：系统**必须**提供聚合**最新** Skill、Rule、学习文章与资讯的首页。**v1 仅展示最新 N 条**，无推荐位与管理员配置精选；后续版本可增加。
- **FR-010**：系统**必须**对认证用户支持管理员角色（管理后台全部内容操作）。**v1 无推荐位管理**。**v1 管理员账号**可通过自注册后手动提升角色，或由种子脚本创建。

### 关键实体

- **Skill（内部）**：独立表。仅元数据：名称、描述、分类、标签、**clone_command**（如 `git clone <GitLab URL>`）、创建者、时间戳；**不存正文与附件**，内容在外部 GitLab 维护。由管理员在后台创建。clone_command 仅校验非空与长度上限（如 2000 字符），不校验格式。详情页提供 clone 命令供复制。无审批状态（发布即生效）。管理员可隐藏或删除。
- **Rule（内部）**：独立表。仅正文（纯文本），无附件。属性：名称、描述、分类、标签、创建者、时间戳。由管理员在后台创建。无审批状态（发布即生效）。管理员可隐藏或删除。
- **外部 Skill**：独立实体（如 ExternalSkill）。仅元数据：名称、描述、安装命令（如 `npx skills add owner/repo`）、来源 URL、可选分类与标签。由管理员添加或通过爬取 [skills.sh](https://skills.sh/) 等录入。详情页提供「安装」引导（复制命令或跳转来源链接）。
- **Category**：用于对 Skill/Rule（及可选学习文章）分类。**由种子数据预置**，不在管理后台维护；含名称与排序；可在 Skill 与 Rule 间共用或按类型区分。
- **Tag**：Skill/Rule 的标签。仅由管理员在后台定义与维护；用于筛选与展示。
- **Learning Article**：标题、摘要、富文本正文、分类；仅由管理员创建与更新。
- **News**：标题、摘要、来源 URL、来源名称、发布日期；仅由管理员创建与更新。
- **User**：登录身份；角色（管理员，用于访问管理后台）。**v1 管理员**可通过种子脚本创建或自注册后手动提升角色。

## 假设

- 认证仅使用账号与密码（v1 无 SSO/OAuth）。
- 平台仅限内部使用（内网）；v1 无公开 SEO 或匿名外网访问要求。
- v1 搜索基于数据库（关键词与分类/标签筛选）；无独立全文检索引擎。
- 设计与布局遵循现有 README 与 styles.css（颜色、排版、布局），与当前静态站保持一致。

## 成功标准 *（必填）*

### 可度量结果

- **SC-001**：未认证用户可在 2 分钟内打开首页、列出 Skill/Rule、应用一次筛选与一次搜索、打开详情页并复制内容。
- **SC-002**：管理员在管理后台提交带必填项的新 Skill 或 Rule 后，30 秒内可在前台公开列表中看到。

- **SC-005**：管理员发布一条学习文章与一条资讯后，保存一分钟内对所有访客可见（无需登录）。
